# 日志系统使用说明

## 功能

现在每个爬取任务都会自动创建独立的日志文件，记录完整的执行过程。

## 日志位置

```
crawl_logs/crawl_{task_id}_{timestamp}.log
```

例如：`crawl_logs/crawl_abc123_20251118_143025.log`

## 查看日志

### 查看最新日志
```bash
# Windows
ls -lt crawl_logs | head -5

# 或者
dir crawl_logs /o-d
```

### 实时监控日志
```bash
# Windows PowerShell
tail -f crawl_logs/crawl_*.log

# 或者查看最新的日志文件
ls crawl_logs -Name | Select-Object -Last 1 | ForEach-Object { tail -f "crawl_logs/$_" }
```

### 搜索错误
```bash
grep "ERROR" crawl_logs/*.log
```

## 日志内容

日志包含：
1. 任务配置信息（URL、深度、模式等）
2. 每个步骤的详细执行过程
3. Playwright 链接提取的详细日志
4. 文章保存和上传的统计信息
5. 错误和警告信息

## 日志级别

- **DEBUG**: 详细的调试信息
- **INFO**: 一般信息（默认显示）
- **WARNING**: 警告信息
- **ERROR**: 错误信息
- **CRITICAL**: 严重错误

## 自动清理

日志文件会一直保留，建议定期手动清理旧日志：

```bash
# 删除7天前的日志
find crawl_logs -name "*.log" -mtime +7 -delete
```

## 问题排查

如果遇到问题，请：
1. 找到对应任务的日志文件
2. 查看完整的执行过程
3. 搜索 "ERROR" 或 "WARNING" 关键词
4. 将相关日志发送给开发人员

## 示例

查看最近一次爬取的完整日志：
```bash
# 找到最新的日志文件
$latest = ls crawl_logs -Name | Select-Object -Last 1

# 查看完整内容
Get-Content "crawl_logs/$latest"

# 或者只看错误
Get-Content "crawl_logs/$latest" | Select-String "ERROR|WARNING"
```
