# 爬取任务日志系统

## 📝 功能说明

已为爬取任务添加详细的日志记录功能，用于调试和追踪爬取过程。

## 📂 日志位置

所有爬取任务的日志保存在 `crawl_logs/` 目录下：

```
crawl_logs/
├── crawl_{task_id}_{timestamp}.log
├── crawl_{task_id}_{timestamp}.log
└── ...
```

## 📊 日志内容

每个日志文件包含：

### 1. 任务配置信息
- 任务ID
- 目标URL
- 爬取深度
- 爬取模式
- 页面限制
- 知识库ID
- 关键词过滤

### 2. 执行过程
- 文章提取过程
- 数据库保存详情
- RAGFlow上传过程
- 每个步骤的成功/失败状态

### 3. 统计信息
- 总文章数
- 新增保存数
- 已存在跳过数
- 空内容跳过数
- 关键词过滤数

### 4. 错误信息
- 详细的错误堆栈
- 警告信息
- 异常处理

## 🔍 日志级别

- **DEBUG**: 详细的调试信息（每篇文章的处理）
- **INFO**: 一般信息（步骤、统计）
- **WARNING**: 警告信息（跳过、未指定等）
- **ERROR**: 错误信息（失败、异常）
- **CRITICAL**: 严重错误（任务崩溃）

## 📖 查看日志

### 方法1：直接查看文件
```bash
# 查看最新的日志
ls -lt crawl_logs/ | head -5

# 查看特定任务的日志
cat crawl_logs/crawl_{task_id}_*.log

# 实时监控日志
tail -f crawl_logs/crawl_{task_id}_*.log
```

### 方法2：搜索关键信息
```bash
# 查找错误
grep "ERROR" crawl_logs/*.log

# 查找RAGFlow上传
grep "RAGFlow" crawl_logs/*.log

# 查找统计信息
grep "统计信息" crawl_logs/*.log
```

## 🎯 重点关注

如果爬取任务没有入库，请检查日志中的：

1. **知识库ID**: 是否正确指定
2. **新增文章数**: 是否为0
3. **RAGFlow上传**: 是否执行，是否成功
4. **错误信息**: 是否有异常

## 🔧 调试步骤

1. 找到对应任务的日志文件
2. 搜索 "步骤" 关键词，查看执行到哪一步
3. 搜索 "ERROR" 或 "WARNING"，查看问题
4. 查看统计信息，确认数据处理情况
5. 检查RAGFlow上传部分的日志

## 🗑️ 清理日志

日志文件会随着时间累积，可以定期清理：

```bash
# 删除7天前的日志
find crawl_logs/ -name "*.log" -mtime +7 -delete

# 删除所有日志
rm -rf crawl_logs/*.log
```

## ⚠️ 注意事项

- 日志文件会占用磁盘空间，建议定期清理
- 调试完成后可以关闭详细日志（修改日志级别）
- 生产环境建议只保留ERROR级别日志
