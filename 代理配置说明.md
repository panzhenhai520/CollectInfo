# 代理配置说明

## 📋 概述

本系统支持灵活的代理配置，可以通过修改 `.env` 文件轻松启用或禁用代理。

## 🔧 配置方式

### 方式1：启用代理（默认）

如果你需要通过代理访问外网（如 Google、Twitter 等），保持以下配置：

```bash
# .env 文件
PROXY_ENABLED=true
PROXY_HTTP=http://127.0.0.1:10808
PROXY_HTTPS=http://127.0.0.1:10808
```

### 方式2：禁用代理

如果你的服务器可以直接访问外网，或者不需要代理，只需修改一行：

```bash
# .env 文件
PROXY_ENABLED=false
# PROXY_HTTP=http://127.0.0.1:10808  # 可以注释掉或保留
# PROXY_HTTPS=http://127.0.0.1:10808
```

**重要**：只需要将 `PROXY_ENABLED` 设置为 `false`，系统会自动忽略所有代理配置。

## 📝 代理配置项说明

| 配置项 | 说明 | 默认值 | 示例 |
|--------|------|--------|------|
| `PROXY_ENABLED` | 是否启用代理 | `true` | `true` 或 `false` |
| `PROXY_HTTP` | HTTP 代理地址 | `http://127.0.0.1:10808` | `http://127.0.0.1:7890` |
| `PROXY_HTTPS` | HTTPS 代理地址 | `http://127.0.0.1:10808` | `http://127.0.0.1:7890` |

## 🔄 修改后如何生效

### 开发环境（本地）

修改 `.env` 文件后，重启应用即可：

```bash
# 停止当前运行的应用（Ctrl+C）
# 然后重新启动
python start_with_schedule.py
```

### 生产环境（服务器）

修改 `.env` 文件后，重启服务：

```bash
# 如果使用 systemd
sudo systemctl restart firecrawl-login

# 或者使用 supervisorctl
sudo supervisorctl restart firecrawl-login
```

## 🌐 代理使用场景

### 需要代理的情况

- 访问被墙的网站（Google、Twitter、Facebook 等）
- 服务器在国内，需要访问国外资源
- 需要隐藏真实 IP

### 不需要代理的情况

- 服务器在国外（如 AWS、GCP、Azure）
- 只爬取国内网站（如香港经济日报、新浪等）
- 服务器已经配置了全局代理

## 🧪 测试代理配置

### 测试代理是否生效

```bash
# 运行测试脚本
python -c "
import config
proxies = config.get_proxies()
if proxies:
    print(f'✅ 代理已启用: {proxies}')
else:
    print('❌ 代理未启用')
"
```

### 测试网络连接

```bash
# 测试是否能访问外网
python -c "
import requests
import config

proxies = config.get_proxies()
try:
    response = requests.get('https://www.google.com', proxies=proxies, timeout=10)
    print(f'✅ 网络连接正常，状态码: {response.status_code}')
except Exception as e:
    print(f'❌ 网络连接失败: {e}')
"
```

## 📌 常见问题

### Q1: 修改配置后不生效？

**A**: 确保重启了应用。配置文件只在应用启动时读取。

### Q2: 代理地址应该填什么？

**A**: 
- **本地开发**：`http://127.0.0.1:端口号`
- **V2rayN**：默认端口 `10808`
- **Clash**：默认端口 `7890`
- **其他代理**：查看代理软件的设置

### Q3: 如何知道代理端口号？

**A**:
- **V2rayN**：右键托盘图标 → 参数设置 → 本地监听端口
- **Clash**：设置 → 端口设置 → HTTP 代理端口
- **命令行查看**：`netstat -ano | findstr "LISTENING"` (Windows)

### Q4: 代理连接失败怎么办？

**A**:
1. 检查代理软件是否正在运行
2. 检查端口号是否正确
3. 尝试禁用代理：`PROXY_ENABLED=false`
4. 查看应用日志：`tail -f app.log`

## 🔍 代码中的代理使用

系统在以下地方使用代理：

1. **requests 库**（HTTP 请求）
   ```python
   import config
   proxies = config.get_proxies()
   response = requests.get(url, proxies=proxies)
   ```

2. **Playwright**（浏览器自动化）
   ```python
   import config
   proxy_config = config.get_playwright_proxy()
   browser = playwright.chromium.launch(proxy=proxy_config)
   ```

3. **Firecrawl API**（如果需要）
   - 自动使用 `config.get_proxies()`

所有代理配置都通过 `config.py` 统一管理，只需修改 `.env` 文件即可。
